# Legado -> SoupReader 迁移文档（迁移级别）

本目录用于把 `/home/server/legado`（Legado）能力与交互语义，按“迁移级别”对齐到 `/home/server/soupreader`（SoupReader）。

## 本轮迁移的明确边界（已决策）

- **排除项处理方式**：仅隐藏入口（UI 不展示），但保留底层模型/迁移键/数据结构。
- **启动阶段交互**：不做强制启动弹窗（隐私协议/首次帮助/更新日志/崩溃提示/首次本地密码等，均只提供设置入口）。
- **压缩包导入**：仅支持 `zip`；`rar/7z` 记为 `blocked` 差异项，保留明确提示与回补计划。

> 说明：SoupReader 已存在集中式迁移排除开关 `lib/core/config/migration_exclusions.dart`，本轮要求把“排除项”落实为 **入口隐藏**（而不是仅 toast blocked）。

## 文档索引

- `00_scope.md`：范围定义、排除项、决策与引用路径
- `01_feature_matrix.md`：功能对照矩阵（唯一验收台账）
- `02_ui_layout_spec.md`：页面级布局排版与交互热区规范
- `03_blocked_log.md`：blocked/差异记录（原因/影响/替代/回补）

## 使用方式（迁移执行约束）

1. **动手改代码前**：先在 `01_feature_matrix.md` 新增/更新条目，并把状态置为“待对齐”或“已存在待核对”。
2. **确认 legado 行为**：对每个条目，必须完整读取 Legado 对应文件后再下结论。
3. **实现与验证**：
   - 实现完成后执行一次 `flutter analyze`（标准：无新增分析错误）。
   - 按矩阵中的“验收步骤”进行手工回归，并回填结果。
4. **结论约束**：在矩阵未逐项对照完成前，禁止使用“完全一致/已一致”等措辞。
